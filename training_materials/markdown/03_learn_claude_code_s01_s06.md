# learn-claude-code 前 6 章学习地图

这里不是逐句复述原文，而是把前 6 章重新整理成“系统如何一层一层长出来”的顺序。

## s01: The Agent Loop

### 这章回答什么问题

模型为什么不等于 Agent。

### 核心观点

- 单次生成只会吐文本
- Agent 必须有主循环
- 工具结果必须重新写回消息历史

### 最小数据结构

- `messages`
- `tool_result`
- `state`

### 对培训最值得讲的一句话

没有 loop，就没有真正能持续推进任务的 Agent。

### 配套代码

- `code/claude_code_s01_s06/s01_agent_loop_minimal.py`

## s02: Tool Use

### 这章回答什么问题

为什么“加一个工具”不应该重写整个循环。

### 核心观点

- 循环不变
- 工具通过 dispatch map 扩展
- 路径安全和工具边界要在 handler 层处理

### 对培训最值得讲的一句话

加工具，本质上是在扩展可触达边界，而不是改 Agent Loop。

### 配套代码

- `code/claude_code_s01_s06/s02_tool_dispatch_minimal.py`

## s03: TodoWrite

### 这章回答什么问题

为什么多步任务会“走着走着就漂了”。

### 核心观点

- 显式计划比隐式记忆更稳
- 同一时间最多一个 `in_progress`
- todo 不是完整任务系统，而是会话内规划

### 对培训最值得讲的一句话

计划不是替模型思考，而是把“现在到底在干什么”写出来。

### 配套代码

- `code/claude_code_s01_s06/s03_todo_manager_minimal.py`

## s04: Subagent

### 这章回答什么问题

为什么大任务不应该把所有中间过程都塞进同一个上下文。

### 核心观点

- 子智能体的核心价值是上下文隔离
- 子任务在独立上下文里完成
- 父智能体只拿回必要总结

### 对培训最值得讲的一句话

子智能体最重要的不是“多一个模型实例”，而是“多一个干净上下文”。

### 配套代码

- `code/claude_code_s01_s06/s04_subagent_minimal.py`

## s05: Skill Loading

### 这章回答什么问题

为什么不能把所有长期知识都永远塞进 system prompt。

### 核心观点

- 先暴露 skill 目录
- 真正需要时再加载 skill 正文
- skill 是按需知识，不是永久提示词肥料

### 对培训最值得讲的一句话

不是把所有知识永远塞进 prompt，而是在需要时加载正确那一份。

### 配套代码

- `code/claude_code_s01_s06/s05_skill_loader_minimal.py`

## s06: Context Compact

### 这章回答什么问题

为什么上下文不是越多越好。

### 核心观点

- 大输出写磁盘，只留预览
- 更旧的结果做微压缩
- 历史过长时做连续性摘要

### 对培训最值得讲的一句话

上下文管理的重点不是“保留全部历史”，而是“保留继续完成任务真正需要的历史”。

### 配套代码

- `code/claude_code_s01_s06/s06_context_compact_minimal.py`

## 前 6 章串起来怎么看

可以把它理解成一个层层递进的系统骨架：

1. `s01` 先建立主循环
2. `s02` 扩展工具边界
3. `s03` 增加会话内规划
4. `s04` 引入上下文隔离
5. `s05` 增加按需知识加载
6. `s06` 解决上下文膨胀问题

如果听众理解了这一串，就已经不再只是“会调用 LLM”，而是开始理解 Agent 系统应该如何逐步长成一个真正可工作的运行时。
