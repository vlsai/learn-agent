# 培训总览

## 这份培训要解决什么问题

这次培训聚焦两条主线：

1. `hello-agents` 第四章里的三种经典智能体构建方式
2. `learn-claude-code` 前 6 章里的核心系统设计思想

目标不是“把资料读完”，而是让听众建立一个足够清晰的工程心智模型：

- 为什么单次调用 LLM 不等于 Agent
- Agent 为什么需要循环、工具、计划、子任务和上下文管理
- 三种经典范式分别更擅长解决什么问题
- 如果自己要做一个教学版 Agent，最小实现应该怎么拆

## 培训对象假设

默认受众是：

- 有一点 Python 基础
- 听说过 LLM / Agent，但没有系统实现过
- 希望快速建立“能讲、能改、能演示”的理解框架

## 推荐培训时长

### 60 分钟版

1. 10 分钟：Agent 为什么不是单次问答
2. 20 分钟：三种经典构建方式
3. 20 分钟：`learn-claude-code` 前 6 章学习地图
4. 10 分钟：代码 demo

### 90 分钟版

1. 15 分钟：概念铺垫
2. 30 分钟：三种范式逐个展开
3. 25 分钟：`learn-claude-code` 前 6 章逐章串讲
4. 20 分钟：现场跑 demo + 提问

## 培训输出

听众离场时，至少应该带走四个结论：

1. 没有 Agent Loop，就没有真正的 Agent
2. 工具调用的关键不是“会不会调”，而是结果能不能重新进入下一轮推理
3. ReAct、Plan-and-Solve、Reflection 三种范式对应的是三种不同的任务组织方式
4. `learn-claude-code` 前 6 章其实是在逐步补齐一个 Agent 系统最核心的骨架
