# 培训总览

## 这份培训要解决什么问题

这次培训不再绑定任何课程章节，只回答三个问题：

1. 三种经典智能体范式分别在解决什么问题
2. 同一个任务换一种工作流，代码为什么会长得不一样
3. 如果要做一套可演示、可复用的最小 Agent demo，应该怎么落地

目标不是“讲很多概念”，而是让听众拿到一个足够清楚的工程心智模型：

- 为什么单次调用 LLM 不等于 Agent
- 为什么工作流组织方式会直接影响结果
- 为什么需要区分“边查边做”“先拆再做”“先做再改”

## 统一 demo 任务

本次培训全部围绕一个目标展开：

> 生成一个单文件贪吃蛇网页。

这个任务很适合培训，因为它同时具备：

- 明确的交互目标
- 明确的成功标准
- 可见的游戏状态变化
- 可以快速验证的运行结果

## 培训对象假设

默认受众是：

- 有一点 Python 基础
- 知道 LLM 是什么，但还没有系统做过 Agent
- 希望建立“能讲、能跑、能改”的理解框架

## 推荐培训时长

### 60 分钟版

1. 10 分钟：为什么要用统一 demo 任务
2. 20 分钟：三种范式讲解
3. 20 分钟：现场跑三种 demo
4. 10 分钟：对比与答疑

### 90 分钟版

1. 15 分钟：Agent 与单次调用的区别
2. 30 分钟：三种范式逐个展开
3. 30 分钟：代码 walkthrough
4. 15 分钟：讨论如何组合落地

## 培训输出

听众离场时，至少应该带走四个结论：

1. 没有循环和状态回流，就没有真正的 Agent
2. ReAct、Plan-and-Solve、Reflection 对应的是三种不同的任务组织方式
3. 同一个任务可以由不同工作流完成，但成本、稳定性、可控性不同
4. 真正落地时，三种范式往往不是互斥，而是可以组合
